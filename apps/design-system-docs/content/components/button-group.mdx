## Overview

A container component for grouping related buttons together. Supports both attached and detached styles, with horizontal and vertical orientations.

<InfoCallout type="info" title="When to Use">
Use button groups to organize related actions or options. Attached groups work well for toggles and segmented controls, while detached groups are ideal for action clusters.
</InfoCallout>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Group related actions">
		Only group buttons that are related in function or context. Each group should represent a logical set of actions.
	</Guideline>
	
	<Guideline type="do" title="Limit group size">
		Keep button groups to 2-5 buttons. Larger groups become difficult to scan and use.
	</Guideline>
	
	<Guideline type="do" title="Use consistent variants">
		Within attached groups, use the same button variant for visual consistency (usually 'secondary').
	</Guideline>
	
	<Guideline type="dont" title="Don't mix button types">
		Don't mix regular buttons and icon buttons in the same attached group. This creates visual inconsistency.
	</Guideline>
	
	<Guideline type="dont" title="Don't use for unrelated actions">
		Don't group buttons that serve completely different purposes just to save space.
	</Guideline>
	
	<Guideline type="dont" title="Don't overuse attached mode">
		Attached button groups work best for toggles and segmented controls. Use detached mode for standard action buttons.
	</Guideline>
</GuidelinesGrid>

## Accessibility

<InfoCallout type="info" title="Keyboard Navigation">
Button groups support standard keyboard navigation. Use Tab to move between buttons and Enter or Space to activate.
</InfoCallout>

- Use `role="group"` for button groups to indicate related controls
- Provide clear labels for icon buttons using `aria-label`
- For toggle groups, use `aria-pressed` to indicate active state
- Ensure keyboard navigation works properly (Tab between buttons)
- Consider using `aria-label` on the group itself to describe its purpose
- Maintain sufficient touch target sizes (minimum 44x44px) on mobile
