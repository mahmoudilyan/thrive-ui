import { Avatar, AvatarImage, AvatarFallback, AvatarStack } from '@thrive/ui';

<Accordions>
  <Accordion title="Overview">
    A flexible avatar component perfect for user profiles, team members, and contact lists. Supports images, fallback initials, status indicators, and notification badges.

    **When to Use:**
    - User profiles and account interfaces
    - Team member lists and directories
    - Comment threads and social features
    - Contact lists and messaging interfaces

    The Avatar component provides graceful fallback from images to initials or icons, ensuring a consistent user experience even when images fail to load.
  </Accordion>
</Accordions>

<Installation />

## Usage

Import the Avatar components from the package:

```tsx
import { Avatar, AvatarImage, AvatarFallback, AvatarStack } from '@thrive/ui';
```

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Provide alt text">
		Always provide descriptive alt text for avatar images to support screen readers.
	</Guideline>
	
	<Guideline type="do" title="Use initials as fallback">
		Provide initials as a fallback when images fail to load or aren't available.
	</Guideline>
	
	<Guideline type="do" title="Choose appropriate sizes">
		Use smaller sizes (xs, sm) for lists and larger sizes (lg, xl) for profile pages or detailed views.
	</Guideline>
	
	<Guideline type="dont" title="Don't omit fallbacks">
		Always provide a fallback (initials or icon) in case the image fails to load.
	</Guideline>
	
	<Guideline type="dont" title="Don't overuse status indicators">
		Only use status indicators when presence information is relevant to the context.
	</Guideline>
	
	<Guideline type="do" title="Use AvatarStack for groups">
		Use AvatarStack component for displaying multiple avatars - it handles overflow automatically.
	</Guideline>
</GuidelinesGrid>

### AvatarStack

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Avatar elements to stack |
| `max` | `number` | `4` | Maximum number of avatars to display before showing overflow |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Size of all avatars in the stack |
| `borderColor` | `'panel' \| 'bg' \| 'transparent'` | `'panel'` | Border color for the ring (adapts to light/dark mode) |
| `className` | `string` | - | Additional className for the container |

## Accessibility

<InfoCallout type="info" title="Image Accessibility">
Always include alt text for avatar images. Use descriptive text like "John Doe's avatar" rather than just "avatar".
</InfoCallout>

- Provide meaningful alt text for all avatar images
- Ensure status colors have sufficient contrast
- Don't rely solely on color to convey status (status dots should be clearly visible)
- Badge counts should be announced by screen readers
- Use semantic HTML and ARIA attributes where appropriate
