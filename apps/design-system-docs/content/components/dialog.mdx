## Overview

A comprehensive dialog system built on Radix UI primitives. Displays content in a modal overlay that requires user interaction before continuing.

<InfoCallout type="info" title="When to Use">
Use dialogs for critical information, confirmations, or forms that require immediate attention. They should interrupt the user's workflow intentionally.
</InfoCallout>

## Basic Usage

<CodeExample
	title="Simple Dialog"
	description="Basic dialog with trigger button"
	code={`<Dialog>

## Controlled Dialog

<CodeExample
	title="Programmatic Control"
	description="Control dialog state programmatically"
	code={`const [open, setOpen] = React.useState(false);

<Dialog open={open} onOpenChange={setOpen}>
  <DialogTrigger asChild>
    <Button>Open</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Controlled Dialog</DialogTitle>
      <DialogDescription>

      </DialogDescription>
    </DialogHeader>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Use for critical actions">
		Dialogs should interrupt the user's flow for important decisions or information that requires immediate attention.
	</Guideline>
	
	<Guideline type="do" title="Provide clear actions">
		Dialog buttons should use action-oriented labels like "Delete", "Save", or "Cancel" rather than "Yes" or "No".
	</Guideline>
	
	<Guideline type="do" title="Keep content focused">
		Dialog content should be concise and focused on a single task or decision.
	</Guideline>
	
	<Guideline type="dont" title="Don't nest dialogs">
		Avoid opening a dialog from within another dialog. This creates a confusing user experience.
	</Guideline>
	
	<Guideline type="dont" title="Don't use for non-critical content">
		Don't use dialogs for content that doesn't require immediate action. Consider using a page or sidebar instead.
	</Guideline>
	
	<Guideline type="dont" title="Don't make them dismissible">
		For critical confirmations, don't allow clicking outside to close. Users should explicitly choose an action.
	</Guideline>
</GuidelinesGrid>

## Accessibility

<InfoCallout type="warning" title="Focus Management">
Dialogs automatically trap focus and return it to the trigger element when closed. Ensure all interactive elements are keyboard accessible.
</InfoCallout>

- Focus is automatically trapped within the dialog
- Pressing Escape closes the dialog (unless disabled)
- Focus returns to the trigger element on close
- Use proper heading hierarchy (DialogTitle as h2)
- Provide descriptive DialogDescription for context
- Ensure keyboard navigation works for all controls
- Don't rely solely on color to convey critical information
- Test with screen readers to ensure proper announcement
