## Overview

A flexible container component for grouping related content and actions. Built with semantic HTML and consistent spacing, perfect for creating organized layouts.

<InfoCallout type="info" title="When to Use">
Use cards to group related information and actions. Cards work well for dashboards, product listings, user profiles, and any content that needs visual separation.
</InfoCallout>

<InfoCallout type="info" title="Flexible Composition">
Cards are built using composition. You can use any combination of CardHeader, CardContent, and CardFooter, or even skip some parts entirely.
</InfoCallout>

## Best Practices

<GuidelinesGrid>
	<Guideline type="do" title="Group related content">
		Use cards to group related information and actions together. Each card should represent a single concept or entity.
	</Guideline>
	
	<Guideline type="do" title="Use semantic structure">
		Follow the CardHeader → CardContent → CardFooter pattern for consistency and accessibility.
	</Guideline>
	
	<Guideline type="do" title="Keep footer actions minimal">
		Limit footer actions to 1-3 buttons. Too many actions can overwhelm users.
	</Guideline>
	
	<Guideline type="dont" title="Don't nest cards deeply">
		Avoid nesting cards within cards. This creates visual confusion and breaks the hierarchy.
	</Guideline>
	
	<Guideline type="dont" title="Don't overload with content">
		Keep card content focused and scannable. If you need more space, consider splitting into multiple cards.
	</Guideline>
	
	<Guideline type="dont" title="Don't skip the header">
		Most cards should have a header with at least a title. Content-only cards should be used sparingly.
	</Guideline>
</GuidelinesGrid>

## Accessibility

<InfoCallout type="info" title="Semantic HTML">
Cards use semantic HTML structure to ensure proper document outline and screen reader support.
</InfoCallout>

- Use semantic HTML structure (article, header, section, footer)
- Provide meaningful titles in CardTitle for context
- Ensure sufficient color contrast for all text
- Make interactive elements keyboard accessible
- Use ARIA labels for icon buttons in card headers
- Consider using `role="region"` with `aria-label` for important cards
