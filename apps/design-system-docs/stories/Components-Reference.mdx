## Usage Guidelines

### Importing Components

```typescript
// From the UI package
import { Button, DataTable, Dialog } from '@thrive/ui';

// From feature components
import { CampaignsList } from '../campaigns/campaigns-list';
import { ContactsList } from '../contacts/contacts-table';
```

### Component Props

All components are fully typed with TypeScript interfaces. Refer to the individual story files for detailed prop documentation.

### Best Practices

- Use semantic component names that reflect their purpose
- Follow the established naming conventions (PascalCase for components)
- Ensure components are accessible and follow WCAG guidelines
- Test components in both light and dark themes
